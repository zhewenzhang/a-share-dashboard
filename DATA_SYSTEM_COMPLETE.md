## 📊 TuShare 频率限制解决方案

### 问题
- TuShare 免费用户每分钟限制 60 次调用
- 高频抓取容易触发限制
- 页面加载慢（每次请求都调用 API）

### 解决方案
| 数据类型 | 更新频率 | 日调用次数 |
|---------|---------|-----------|
| 指数数据 | 5 分钟 | ~288 次 |
| 板块数据 | 15 分钟 | ~96 次 |
| 股票数据 | 30 分钟 | ~48 次 |
| **总计** | - | **~432 次/天** |

**优势：**
- ✅ 远低于 TuShare 限制（60 次/分钟）
- ✅ 页面响应速度提升 10-100 倍
- ✅ 支持离线访问（使用缓存）
- ✅ 自动更新，无需手动刷新

## 🚀 使用说明

### 1. 启动后端服务
```bash
cd /Users/dave/a-share-dashboard/backend
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

启动后会自动：
- 创建 SQLite 数据库
- 启动定时任务调度器
- 预加载数据

### 2. 访问前端页面
```
http://localhost:8080
```

页面会自动：
- 检查 API 连接状态
- 加载指数数据
- 加载板块数据
- 加载推荐股票
- 显示更新时间

### 3. 刷新数据
点击页面上的 **"刷新"** 按钮，会重新加载所有数据。
